JavaScript Datatyper

Number

Bruges til at repræsentere både hele tal og decimaler.

let age = 30;  // Heltal
let price = 19.99; // Decimaltal

BigInt

Bruges til at håndtere meget store heltal, som ikke kan repræsenteres af standard Number.

let bigNum = 1234567890123456789012345678901234567890n;

String

Bruges til tekst og kan oprettes med enten enkelt-, dobbelt- eller backticks (`).

let message = "Hej, verden!";
let greeting = 'Velkommen!';
let name = `Navn: Alice`;

Boolean

Repræsenterer to mulige værdier: true (sand) og false (falsk). Bruges ofte i betingelser og loops.

let isActive = true;
let isComplete = false;

Null

Angiver, at en variabel bevidst er uden værdi. Det betyder, at der ikke er nogen data knyttet til den.

let person = null;  // Ingen værdi

Undefined

En variabel, der er erklæret, men ikke tildelt en værdi, har værdien undefined.

let user;
console.log(user); // undefined

Object

Bruges til at gemme samlinger af data. Et objekt kan indeholde flere nøgle-værdi-par.

let person = { name: "Alice", age: 25 };

Symbol

En unik og uforanderlig værdi, som ofte bruges til at skabe unikke identifikatorer.

let uniqueId = Symbol("id");

Automatisk og Tvangs-konvertering i JavaScript

1. Automatisk konvertering

Når du bruger + mellem en streng og et tal, tvinges tallet til at blive en streng: 

let result = "The number is " + 5;  // JavaScript konverterer 5 til en streng, så resultatet bliver "The number is 5"

2. Tvinge konvertering

Med funktioner som Number(), kan du tvinge en værdi til en bestemt datatype: 

let num = "123"; 
let convertedNum = Number(num); // '123' bliver nu et tal (123)

Hvis du prøver at konvertere noget, der ikke er et tal, som f.eks. "abc", vil det returnere NaN (Not a Number).

3. Konvertering af falske værdier

JavaScript betragter bestemte værdier som "falske" i betingelser, hvilket betyder, at de automatisk konverteres til false: 

if (!null) console.log("null er falsk");  // Dette bliver logget, fordi null konverteres til false

Værdier som null, undefined, en tom streng "", 0, og NaN betragtes som falske, og derfor udløser de betingelsen i if-sætningen.
