Hvad er Javascript?

Hvad er JavaScript?

JavaScript blev oprindeligt skabt for at "gøre websider levende". Programmer i dette sprog kaldes scripts. De kan skrives direkte i en websides HTML og køres automatisk, når siden indlæses.

Scripts leveres og udføres som almindelig tekst. De behøver ikke en speciel forberedelse eller kompilering for at køre.

På denne måde adskiller JavaScript sig fra et andet sprog kaldet Java.

Hvorfor JavaScript?

Da JavaScript blev skabt, havde det et andet navn: "LiveScript". Men Java var meget populært på det tidspunkt, så det blev besluttet at positionere det nye sprog som en "lillebror" til Java. Men med tiden udviklede JavaScript sig til et helt selvstændigt sprog med sin egen specifikation kaldet ECMAScript, og nu har det ikke direkte relation til Java.

I dag kan JavaScript køre ikke kun i browsere, men også på serveren eller faktisk på enhver enhed, der har et specielt program kaldet JavaScript-Engine.

Browsere har en indbygget motor, der undertiden kaldes "JavaScript-virtualmaskinen".

Forskellige Engines har forskellige navne. For eksempel:

  • V8 – i Chrome og Opera.
  • SpiderMonkey – i Firefox.

…Der er andre kodenavne som "Trident" og "Chakra" for forskellige versioner af IE, "ChakraCore" for Microsoft Edge, "Nitro" og "SquirrelFish" for Safari, osv.

Disse udtryk er gode at huske, fordi de bruges i udviklerartikler på internettet. Vi vil også bruge dem. For eksempel, hvis "en funktion X understøttes af V8", så virker den sandsynligvis i Chrome og Opera.

Hvad kan JavaScript i browseren?

Moderne JavaScript er et "sikkert" programmeringssprog. Det giver ikke lavniveauadgang til hukommelse eller CPU, fordi det oprindeligt blev skabt til browsere, som ikke kræver det.

JavaScripts muligheder afhænger meget af det miljø, det kører i. For eksempel kan Node.js understøtte funktioner, der giver JavaScript mulighed for at læse/skrive vilkårlige filer, udføre netværksanmodninger, osv.

I browseren kan JavaScript gøre alt, der relaterer til webside-manipulation, interaktion med brugeren og webserveren.

For eksempel kan JavaScript i browseren:

  • Tilføje ny HTML til siden, ændre eksisterende indhold, ændre stilarter.
  • Reagere på brugerhandlinger, køre ved museklik, pointerbevægelser, tastetryk.
  • Sende anmodninger over netværket til fjernservere, downloade og uploade filer (såkaldte AJAX og COMET teknologier).
  • Få og indstille cookies, stille spørgsmål til besøgenden, vise beskeder.
  • Husk data på klientsiden ("local storage").

Hvad kan JavaScript IKKE i browseren?

JavaScripts muligheder i browseren er begrænsede for brugerens sikkerhed. Formålet er at forhindre en ondsindet webside i at få adgang til private data eller skade brugerens data.

Eksempler på sådanne begrænsninger inkluderer:

  • JavaScript på en webside kan måske ikke læse/skrive vilkårlige filer på harddisken, kopiere dem eller køre programmer. Det har ikke direkte adgang til OS-systemfunktioner.
  • Moderne browsere tillader det at arbejde med filer, men adgangen er begrænset og kun tilgængelig, hvis brugeren udfører specifikke handlinger som at "droppe" en fil i et browser-vindue eller vælge den via en input-tag.
  • Der er måder at interagere med kamera/mikrofon og andre enheder på, men de kræver brugerens eksplicitte tilladelse. Så en JavaScript-aktiveret side kan ikke slå kameraet til i smug og observere omgivelserne.
  • Forskellige faner/vinduer kender normalt ikke til hinanden. Undtagelser inkluderer vinduer, hvor JavaScript fra den ene side åbner den anden. Men selv i dette tilfælde kan JavaScript fra den ene side måske ikke få adgang til den anden, hvis de kommer fra forskellige websteder (forskellige domæner, protokoller eller porte).

Dette kaldes "Same Origin Policy". For at omgå dette skal begge sider acceptere det og indeholde speciel kode, der håndterer dataudveksling. Vi vil dække dette senere i tutorialen.

JavaScript kan nemt kommunikere over nettet til den server, hvor den aktuelle side kommer fra. Men dens evne til at modtage data fra andre websteder/domæner er begrænsede. Selvom det er muligt, kræver det en protokol aftale ( HTTP) fra den eksterne side.

Sådanne begrænsninger eksisterer ikke, hvis JavaScript bruges uden for browseren, for eksempel på en server. Moderne browsere tillader også plugins/udvidelser, der kan anmode om udvidede tilladelser.

Hvad gør JavaScript unikt?

Der er mindst tre gode ting ved JavaScript:

  • Fuldt integration med HTML/CSS.
  • ting gøres enkelt.
  • Understøttet af alle større browsere og slået til som standard.

JavaScript er det eneste browser-teknologi, der kombinerer disse tre ting.

Det er det, der gør JavaScript unikt. Det er derfor, det er det mest udbredte værktøj til at skabe browser-grænseflader.

"Sprog over" JavaScript

Så for nylig er en masse nye sprog dukket op, som transpileres (konverteres) til JavaScript, før de kører i browseren.

Moderne værktøjer gør transpileringen meget hurtig, hvilket faktisk tillader udviklere at kode på et andet sprog og konvertere det automatisk.

Eksempler på sådanne sprog:

  • CoffeeScript er en "syntaktisk sukker" til JavaScript. Det introducerer kortere syntaks, hvilket giver os mulighed for at skrive mere præcis og klar kode. Normalt kan Ruby-udviklere lide det.
  • TypeScript fokuserer på at tilføje "stærk datatypering" for at forenkle udviklingen og understøttelsen af komplekse systemer. Det er udviklet af Microsoft.
  • Flow tilføjer også datatyper, men på en anden måde. Udviklet af Facebook.
  • Dart er et selvstændigt sprog, der har sin egen motor, der kører i ikke-browser-miljøer (som mobilapps), men også kan transpileres til JavaScript. Udviklet af Google.

Der er flere. Selvfølgelig, selv hvis vi bruger et af de transpilerede sprog, bør vi også kende JavaScript for virkelig at forstå, hvad der foregår.

Resumé

JavaScript blev oprindeligt skabt som et browsersprog, men bruges nu i mange andre miljøer.

I dag har JavaScript en unik position som det mest udbredte browser-sprog med fuld integration med HTML/CSS.

Der er mange sprog, der "transpileres" til JavaScript og giver visse funktioner. Det anbefales at tage et kig på dem, i det mindste kort, efter at have mestret JavaScript.
