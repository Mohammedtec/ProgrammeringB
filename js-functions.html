Javascript funktioner

En JavaScript-funktion er en blok af kode, der er designet til at udføre en bestemt opgave. Funktioner kan tage input (kaldet parametre eller argumenter), udføre handlinger baseret på dette input og returnere et resultat. Funktioner er en grundlæggende byggesten i JavaScript og bruges til at organisere kode, gøre den genanvendelig og let at vedligeholde. Her er et simpelt eksempel på en JavaScript-funktion:.

Vi har allerede set eksempler på indbyggede funktioner som alert(message), prompt(message, default)og confirm(question). Men vi kan også lave vores egne funktioner.

Funktionserklæring

For at oprette en funktion kan vi bruge en funktionserklæring . Det ser sådan ud:

function showMessage() {
alert( 'Hello everyone!' );
}

Når erklærer en funktion så vil der ikke ske noget, den bliver blot oprettet i hukomelsen.
for at kør koden skal man call funktionen ved i dette tilfælde bruge Showmessage();

Lokal Variable

Når man laver en funktion kan en variable kun læses i selve funktionen,

function showMessage() {
let message = "Hello, I'm JavaScript!"; // local variable
alert( message );
}
showMessage(); // Hello, I'm JavaScript!
alert( message ); // Error! The variable is local to the function

som man kan se på koden, så har vi en variable som hedder message, vi laver en alert, så at vi kan vise teksten i vores variable, og dette vises først når vi kalder på funktionen showMessage, Når vi prøver at lave en alert besked udenfor funktionen, så vil den vise en fejl.

Global Variable

En global variable er en variable som kan læses i hele programmet,

let userName = 'John';
function showMessage() {
let message = 'Hello, ' + userName;
alert(message);
}
showMessage(); // Hello, John

på kode eksempelet kan man se en glbal variable userName med værdien john, den kan tilgåes af hele programmet, som kan ses i koden så henter vi userName og sætter den sammen med message, som viser begge værdier når vi kalder på funktionen

Her kan vi se hvordan vi kan ændre variablens værdi i funktionen. I funktionen ændre vi værdien af userName fra john til bob. john er global værdi og kan læses af hele programmet, som gør at vi kan ændre værdien af userName i funktionen. Når vi alert username for anden gang så ser vi at den har ændrede værdi til Bob.

let userName = 'John';
function showMessage() {
userName = "Bob"; // (1) changed the outer variable
let message = 'Hello, ' + userName;
alert(message);
}
alert( userName ); // John before the function call
showMessage();
alert( userName ); // Bob, the value was modified by the function

Parameter

I denne kodeeksempel ser vi, hvordan funktioner kan tage parametre for at gøre dem mere fleksible og genanvendelige.

function showMessage(from, text) { // parameters: from, text
alert(from + ': ' + text); }
showMessage('Ann', 'Hello!');
showMessage('Ann', "What's up?");


Funktionens navn: showMessage Parametre: from og text – de fungerer som lokale variabler inde i funktionen. Inde i funktionen bruger vi alert() til at vise en besked. Kombinering af tekst: from + ': ' + text sammensætter strenge ved at bruge +-operatoren.

Første kald: "Ann" bliver sat som from, og "Hello!" som text, så beskeden der vises er: Ann: Hello!

Andet kald: "Ann" er stadig from, men "What's up?" er nu text, så beskeden bliver: Ann: What's up?

default værdier i funktioner

Når showMessage('Ann', 'Hello!') kaldes, bliver værdierne "Ann" og "Hello!" kopieret til parametrene from og text. Inde i funktionen bruges disse værdier til at danne beskeden.

I JavaScript kan vi angive standardværdier for funktioners parametre. Hvis en funktion kaldes uden en værdi for en parameter, bliver den undefined. Men vi kan give parametre standardværdier, så de får en foruddefineret værdi, hvis ingen værdi gives.

Eksempel 1: Manglende parameter bliver undefined

function showMessage(from, text) {
alert(from + ": " + text);
}
showMessage("Ann"); // Ann: undefined

Funktionen showMessage tager to parametre: from og text. Når vi kalder showMessage("Ann"), mangler argumentet for text. Resultat: text bliver undefined, og output bliver: Anne:Undifined

Eksempel 2: Tilføjelse af en standardværdi

function showMessage(from, text = "no text given") {
alert(from + ": " + text);
}
showMessage("Ann"); // Ann: no text given

Vi angiver en default-værdi for text ved at skrive text = "no text given". Hvis text ikke er angivet, bruges "no text given".

Eksempel 3: Hvis værdien er undefined

showMessage("Ann", undefined); // Ann: no text given

Når vi eksplicit sender undefined som argument, træder standardværdien i kraft. så får vi resultat Ann: no text given

Eksempel 4: Brug af en funktion som standardværdi

function anotherFunction() {
return "Generated text";
}
function showMessage(from, text = anotherFunction()) {
alert(from + ": " + text);
}
showMessage("Ann"); // Ann: Generated text

Hvis text ikke gives, kalder JavaScript anotherFunction(), som returnerer "Generated text".

Alternative standardparametre

Nogle gange giver det mening at tildele standardværdier for parametre på et senere tidspunkt efter funktionserklæringen. Vi kan kontrollere, om parameteren er vidergivet under funktionsudførelsen, ved at sammenligne den med undefined:

function showMessage(text) {
// ... if (text === undefined) { // if the parameter is missing
text = 'empty message';
}
alert(text);
}
showMessage(); // empty message

Eksempler på funktioner med passende navne:

Funktioner repræsenterer handlinger, så deres navn er normalt et udsagnsord. Navnet bør være kort, præcist og beskrive, hvad funktionen gør, så enhver, der læser koden, hurtigt kan forstå dens formål.
Det er en almindelig praksis at starte en funktion med et ord, der groft beskriver handlingen.
Almindelige præfikser til funktionsnavne:
"show…" → viser noget
"get…" → returnerer en værdi
"calc…" → beregner noget
"create…" → opretter noget
"check…" → tjekker en betingelse og returnerer true/false

Eksempler på funktioner med passende navne:
showMessage(..) // viser en besked
getAge(..) // returnerer en alder (henter den på en eller anden måde)
calcSum(..) // beregner en sum og returnerer resultatet
createForm(..) // opretter en formular (og returnerer den typisk)
checkPermission(..) // tjekker en tilladelse, returnerer true/false

calback function

Callback-funktioner Callback-funktioner er en grundlæggende del af JavaScript og mange andre programmeringssprog. De giver dig mulighed for at sende funktioner som argumenter til andre funktioner og udføre dem senere, ofte som svar på en begivenhed eller en betingelse.

I dette eksempel har vi defineret tre funktioner: myDisplayer, myFirst og mySecond. Disse funktioner bruges til at vise en besked i et HTML-element med id'et demo

function myDisplayer(some) {
document.getElementById("demo").innerHTML = some;
}
function myCalculator(num1, num2, myCallback) {
let sum = num1 + num2;
myCallback(sum); // Kalder callback-funktionen med resultatet
}
myCalculator(5, 5, myDisplayer); // Her sendes `myDisplayer` som en callback

Forklaring:
1. myCalculator funktionen tager tre parametre: num1, num2 og myCallback.
2. Den beregner summen af num1 og num2.
3. Derefter kalder den myCallback-funktionen og sender summen som argument.
4. myDisplayer funktionen viser resultatet i HTML-elementet med id'et demo.

Callbacks er nyttige, når du har brug for at udføre en handling efter at en anden handling er fuldført. De er især vigtige i asynkrone situationer, f.eks.: API-kald: Vent på data fra en server, og udfør derefter en handling. Timere: Udfør en handling efter en forsinkelse med setTimeout. Filindlæsning: Vent på, at en fil er indlæst, og udfør derefter en handling.
